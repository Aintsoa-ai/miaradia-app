   - Fixed the issue where Chrome blocked programmatic `.showPicker()` calls because React Native Web's synthetic event wrappers broke the user gesture token.
   - Designed a click-through pointer-events system:
     * Set `pointerEvents: 'none'`, `width: '1px'`, `height: '1px'`, and `zIndex: -1` on the hidden Web inputs (`<input type="date">` and `<input type="datetime-local">`).
     * This forces the user's click to pass right through the invisible input and hit the container `div` or its contents (icon, text).
   - Created a global `window` event listener inside a `useEffect` hook in `app/(tabs)/index.tsx` and `app/(tabs)/publish.tsx`.
   - The listener is registered with `{ capture: true }` to run in the browser's native synchronous event dispatch phase before React Native intercepts or bubbles it.
   - When a click on a container with `data-date-picker-trigger` is intercepted, it synchronously triggers `.showPicker()` on the hidden input.
   - Chrome registers this as a trusted user gesture and opens the calendar popup instantly when clicking anywhere on the box (the label, the icon, or the container).
